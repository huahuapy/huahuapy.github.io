vue高级（三）

发表于 2021-01-15 | 分类于 vue
Vue响应式

  • 特点

    • 数据驱动视图的第一步，数据data变化，立即触发视图的更新
    • 核心API：Object.defineProperty
    • Vue3.0已经启用了proxy（兼容性不好，无法用polyfill）

  • 功能

    》监听对象，监听数组
    》深度监听
    》几个缺点

    总的来说， Object.defineProperty在深度监听时需要递归，一次性计算量比较大

    另外，对于新增和删除的属性，无法监听到（所以需要Vue.set以及Vue.delete）

    对于数组，Object.defineProperty无法监听原生数组，需要特殊处理

VDom和diff算法

  • 简介

    • Dom计算耗费性能——> 先用JS模拟Dom结构，计算出最小的变更，然后操作Dom
    • vDom———参考snabbdom
    • diff即对比，不同于树diff时间复杂度在O(n3)， 这里的时间复杂度优化到了O(n)。具体做法就是：只比较同一层，不跨层比较； 如果tag不同，就删掉重建； tag和key两者都相同的话就认为是相同

  • diff算法要点

    • patchVnode（vnode渲染出来）
    • addVnodes removeVnodes
    • updateChildren(key很重要)更新
模板编译

  • vue template compiler将模板编译为render函数

  • 执行render生成Vnode

  • 组件更新和渲染的过程

    初次渲染

    • 解析模板为render函数，或者通过webpack(vue-loader)在开发环境下已经完成（有时候自己写一些小demo可能是用vue-template-compiler等，在浏览器中完成的）
    • 触发响应式，监听data（getter， setter）
    • 执行render函数，生成vnode，patch(elem, vnode)

    更新

    • 修改data，触发setter
    • 重新执行render， 生成newVnode
    • patch(vnode, newVnode)

    异步渲染

    • nextTick
    • 汇总渲染，一次性渲染， 可以提高性能，减少了Dom渲染次数

生命周期

挂载——>更新——>销毁

1、beforeCreate（创建前）

表示实例完全被创建出来之前，vue 实例的挂载元素$el和数据对象 data 都为 undefined，还未初始化。

2、created（创建后）

数据对象 data 已存在，可以调用 methods 中的方法，操作 data 中的数据，但 dom 未生成，$el 未存在 。

3、beforeMount（挂载前）

vue 实例的 $el 和 data 都已初始化，挂载之前为虚拟的 dom节点，模板已经在内存中编辑完成了，但是尚未把模板渲染到页面中。data.message 未替换。

4、mounted（挂载后）

vue 实例挂载完成，data.message 成功渲染。内存中的模板，已经真实的挂载到了页面中，用户已经可以看到渲染好的页面了。实例创建期间的最后一个生命周期函数，当执行完 mounted 就表示，实例已经被完全创建好了，DOM 渲染在 mounted 中就已经完成了。

5、beforeUpdate（更新前）

当 data 变化时，会触发beforeUpdate方法 。data 数据尚未和最新的数据保持同步。

6、updated（更新后）

当 data 变化时，会触发 updated 方法。页面和 data 数据已经保持同步了。

7、beforeDestory（销毁前）

组件销毁之前调用 ，在这一步，实例仍然完全可用。

8、destoryed（销毁后）

组件销毁之后调用，对 data 的改变不会再触发周期函数，vue 实例已解除事件监听和 dom绑定，但 dom 结构依然存在。

​ 如果使用的是以下import的引入方式，就是同步引入， 页面加载时一定会同时也加载这个组件

​ 这种异步组件的引入方式可以使组件只在使用时才被加载，不需要使用的话就不加载
