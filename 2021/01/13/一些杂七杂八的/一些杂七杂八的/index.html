hash

路由中的hash路由方面：

  • hash不刷新页面，只跳转页面，SPA
  • hash永远不会提交到后端，前端自己搞定
  • hash的改变有几种方式：
    • JS代码改变URL
    • 手动改变URL中的hash
    • 前进后退

history

hash路由刷新（刷新一般是server参与）,后端一般在刷新时返回index.html, hash前端自己搞定，所以刷新后仍是这个页面。

history是另一种路由模式，在刷新后会404，需要后端支持。

history模式的实现与监听则是主要由history.pushState以及window.onpopstate完成路由改变。

v-show & v-if

v-show是利用css中display:none实现的

v-if是组件的渲染和销毁来达到显示和隐藏

所以如果频繁切换还是用v-show

为什么v-for中用key

因为在diff算法中是通过key与tag来判断是不是sameNode, 如果用index或者是random可能引起混乱且不如用key能够减少渲染的次数, 提高渲染的性能

生命周期

挂载——>更新——>销毁

1、beforeCreate（创建前）

表示实例完全被创建出来之前，vue 实例的挂载元素$el和数据对象 data 都为 undefined，还未初始化。

2、created（创建后）

数据对象 data 已存在，可以调用 methods 中的方法，操作 data 中的数据，但 dom 未生成，$el 未存在 。

3、beforeMount（挂载前）

vue 实例的 $el 和 data 都已初始化，挂载之前为虚拟的 dom节点，模板已经在内存中编辑完成了，但是尚未把模板渲染到页面中。data.message 未替换。

4、mounted（挂载后）

vue 实例挂载完成，data.message 成功渲染。内存中的模板，已经真实的挂载到了页面中，用户已经可以看到渲染好的页面了。实例创建期间的最后一个生命周期函数，当执行完 mounted 就表示，实例已经被完全创建好了，DOM 渲染在 mounted 中就已经完成了。

5、beforeUpdate（更新前）

当 data 变化时，会触发beforeUpdate方法 。data 数据尚未和最新的数据保持同步。

6、updated（更新后）

当 data 变化时，会触发 updated 方法。页面和 data 数据已经保持同步了。

7、beforeDestory（销毁前）

组件销毁之前调用 ，在这一步，实例仍然完全可用。

8、destoryed（销毁后）

组件销毁之后调用，对 data 的改变不会再触发周期函数，vue 实例已解除事件监听和 dom绑定，但 dom 结构依然存在。
